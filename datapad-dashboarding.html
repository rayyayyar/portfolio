DataPad Dashboarding

DataPad, a web app designed for teams to integrate, refine, analyze, explore, and share data.

Building dashboarding tools remains another challenging design problem frequent in data visualization and BI. Our most basic question was how do we help users put together a collection of analyses from various datasets and projects? Additional problems, listed below, were also part of the process.

DataPad's dashboarding tool aimed to be flexible, explicit in conveying where a chart came from, and simple with global filtering.

Viewing a SaaS Dashboard. A completed dashboard in view mode allows for filtering and device-type preview.

The Initial Prototype Used Preset Layout Templates

The "Edges" Concept Uses a Docking Model

DataPad's vision for a dashboarding tool sought to balance intuitivenes with the flexibility of creating custom layouts.

The "Edges" Docking Model Dashboard. A docking model would help users quickly create layouts without having to manually move and resize.

The "Canvas Grid" Concept Affords Better Resizing

The "Canvas Grid" Dashboard. While extremely flexible and easy to understand, this would place much more burden on the user to move and resize panes to create useful dashboard layouts.
